# 移动端适配体验部分修改计划

## 1. 现状分析
当前代码库中已基本搭建了 `Experience.tsx`（体验首页）和 `ModelWorkspace.tsx`（模型工作台）的移动端基础结构。
在 `ModelWorkspace.tsx` 中，已经通过 `isVision` 变量区分了**视觉模型**和**语言模型**的两种不同排版（并列存在），但细节上需要根据提供的PC端截图要求进行进一步的移动端适配和体验优化：
- **模型分类**：`Experience.tsx` 中仍存在“图片”和“视频”分类，未统一为“视觉模型”。
- **视觉模型适配**：PC端图2的排版在移动端已部分实现，但需要确保“素材资产(Library)”与“模型参数(Settings)”图标在移动端顶部导航栏位置统一，并且仅在视觉模型下展示素材资产。
- **语言模型适配**：PC端图1的“图像理解任务”示例卡片使用的是真实的背景图片，而当前移动端代码仅使用了基础的SVG Icon，视觉效果不符，需统一适配为带真实背景图的移动端滚动卡片样式。

## 2. 实施步骤

### 步骤一：合并图像和视频分类 (Experience.tsx)
- **修改分类定义**：在 `categories` 数组中，将 `{ title: '图片' }` 和 `{ title: '视频' }` 合并为 `{ title: '视觉' }`。
- **更新列表渲染逻辑**：在 `renderModels` 函数中，增加当 `activeTab === '视觉'` 时的判断逻辑，筛选并展示所有的视觉类模型（例如 `Doubao-Seedream-4.5` 和 `Doubao-Seedance-1.5-pro`），彻底实现“不再区分图像和视频分类”。

### 步骤二：统一模型参数与素材资产位置 (ModelWorkspace.tsx)
- **头部导航栏优化**：在 `ModelWorkspace.tsx` 的 Header 部分，已经有 `Library`（素材资产）和 `Settings2`（模型参数）图标并列放置。
- **动态展示**：增加逻辑控制，使 `Library`（素材资产）图标仅在 `isVision === true`（即视觉模型）时展示，同时保持与 `Settings2` 图标位置统一紧凑，符合移动端操作习惯。

### 步骤三：语言模型示例卡片视觉升级 (ModelWorkspace.tsx)
- **替换 Icon 为真实图片**：在语言模型的“试试以下图像理解任务”区域，将原本纯色背景 + SVG Icon 的卡片样式（如 `进行图像分类`、`生成图片标题` 等），替换为与 PC 端（图1）一致的真实背景图样式。
- **移动端样式适配**：使用真实图片（可使用 placeholder 占位图如 `https://picsum.photos/...` 模拟效果）作为卡片背景或主体，增加遮罩或文字底部渐变效果，使其在横向滚动的移动端容器中展示更加美观，与 PC 端体验对齐。

## 3. 验证步骤
- 打开 `/console/experience` 页面，确认 Tab 栏中“图片”和“视频”已合并为“视觉”，点击“视觉”能正常显示图像和视频生成模型。
- 点击视觉模型进入工作台，确认顶部右侧同时显示“素材库(Library)”和“参数设置”图标，位置统一。
- 返回并点击语言模型进入工作台，确认顶部右侧仅显示“参数设置”图标；同时下方“图像理解任务”示例卡片已经变为真实的图片背景排版，且在移动端可正常横向滑动。
